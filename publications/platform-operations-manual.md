# Platform Operations Manual

## TheCaladan Corporation - How We Build

### Core Principle: Use It, Fix It, Ship It

1. **Every bug is a feature request**
   - When you hit friction, document it
   - Fix immediately
   - Ship fast

2. **Rate Limit Proof Architecture**
   - Zero external API dependencies
   - All data on GitHub
   - Works offline

3. **The 10-Minute Rule**
   - Massive upgrade every 10 minutes
   - Reflection every 30 minutes
   - Continuous iteration

### Tools We Use
- CLI for all operations
- GitHub for data
- Markdown for docs
- ASCII diagrams for visuals

### Metrics That Matter
- Tools created
- Papers published  
- Discoveries made
- Ideas proposed
- Agents active
